# Как подключить AI-разбор брифа (YandexGPT) — пошагово

**Что мы строим.** Пользователь вставляет бриф → маленький «переходник» (облачная функция в Яндекс Облаке) отправляет текст в YandexGPT → получаем этапы и часы → калькулятор считает сумму сам.
**Важно:** данные не покидают РФ, персональные данные вырезаются, ключи хранить не нужно.

**Что понадобится:** карта РФ (для Яндекс Облака — новичкам дают бесплатный грант), ~40 минут.
**Файл с кодом функции:** `cloud-function/index.js` (лежит рядом с этой инструкцией).

> Названия кнопок в консоли Яндекса могут чуть отличаться — ориентируйся по смыслу.

---

## Часть 1. Доступ к YandexGPT

### Шаг 1. Регистрация в Яндекс Облаке
1. Открой https://console.yandex.cloud
2. Войди под своим Яндекс-аккаунтом (или создай).
3. Согласись с условиями.

### Шаг 2. Платёжный аккаунт
1. Слева выбери **Billing (Платежи)** → **Создать платёжный аккаунт**.
2. Привяжи карту. Спишется 1 ₽ для проверки и вернётся. Новым пользователям обычно дают пробный грант.

### Шаг 3. Узнать ID каталога (folder)
1. На главной странице консоли выбери своё облако.
2. Внутри будет каталог (по умолчанию `default`).
3. Открой его и **скопируй его ID** — строка вида `b1g xxxxxxxxxxxx`.
   👉 Запиши куда-нибудь, назовём его **FOLDER_ID**.

### Шаг 4. Включить YandexGPT (Foundation Models / AI Studio)
1. В поиске сверху набери **Foundation Models** (или **AI Studio**) и открой.
2. Нажми **Подключить / Активировать**, прими условия использования YandexGPT.
   ✅ После этого модель доступна в твоём каталоге.

### Шаг 5. Сервисный аккаунт (чтобы функция могла обращаться к модели без ключа)
1. Слева **Identity and Access Management (IAM)** → **Сервисные аккаунты** → **Создать**.
2. Имя, например: `ai-brief-sa`.
3. Дай ему роль: нажми **Добавить роль** → найди и выбери **`ai.languageModels.user`**.
4. Сохрани.
   ✅ Ключ создавать НЕ нужно — функция получит доступ автоматически.

---

## Часть 2. Прокси = облачная функция

### Шаг 6. Создать функцию
1. В поиске набери **Cloud Functions (Функции)** → открой → **Создать функцию**.
2. Имя: `ai-brief-proxy`. Создать.

### Шаг 7. Вставить код
1. Внутри функции нажми **Создать редактор / Создать версию**.
2. Среда выполнения (Runtime): **Node.js 22**.
3. Открой файл `cloud-function/index.js` (рядом с этой инструкцией), скопируй ВСЁ содержимое и вставь в файл `index.js` в редакторе.

### Шаг 8. Настройки версии
Прямо на экране создания версии заполни:
1. **Точка входа (Entry point):** `index.handler`
2. **Таймаут:** 30 секунд.
3. **Сервисный аккаунт:** выбери `ai-brief-sa` (из Шага 5).
4. **Переменные окружения** → добавь:
   - `FOLDER_ID` = твой FOLDER_ID из Шага 3
   - *(необязательно)* `YAGPT_MODEL` = `yandexgpt` (Pro) или `yandexgpt-lite` (дешевле)
   - `ALLOWED_ORIGINS` = `https://desidoc.ru`
5. Нажми **Создать версию** (задеплоится за несколько секунд).

### Шаг 9. Сделать функцию публичной и взять ссылку
1. Вкладка **Обзор** функции → включи **Публичная функция** (Public).
2. Скопируй **ссылку для вызова** — вид `https://functions.yandexcloud.net/d4xxxxxxxxxxxx`.
   👉 Назовём её **URL_ФУНКЦИИ**.

### Шаг 10. Проверить, что работает (без сайта)
1. Вкладка **Тестирование** → в теле запроса вставь:
   ```json
   { "brief": "Лендинг для кофейни, 5 экранов, нужен дизайн и графики" }
   ```
2. Запусти. В ответе должен прийти JSON с полем `stages` (этапы и часы).
   ✅ Если пришёл JSON — прокси работает.
   ⚠️ Если ошибка «нет токена» — проверь, что привязан сервисный аккаунт (Шаг 8.3).
   ⚠️ Если «AI недоступен» — проверь FOLDER_ID и что YandexGPT активирован (Шаг 4).

---

## Часть 3. Подключить сайт

Файл настройки: `js/ai-config.js`.

### Шаг 11. Вставить URL функции в сайт
Код сайта уже подготовлен: он вызывает YandexGPT через Cloud Function и сохраняет локальный fallback при временной ошибке.

1. Открой файл `js/ai-config.js`.
2. Вставь URL функции между кавычками:

```js
window.DESIDOC_AI_BRIEF_ENDPOINT = "https://functions.yandexcloud.net/dXXXXXXXX";
```

3. Опубликуй обновлённый файл вместе с остальными файлами сайта. API-ключ или IAM-токен в сайт не вставляй.

---

## Часть 4. Согласие и политика (обязательно)

### Шаг 12. Предупреждение в окне брифа
В окне ввода брифа покажи текст (можно через существующий `setBriefAiStatus`):
> «Текст обрабатывается ИИ для оценки сметы. Не вставляйте персональные данные третьих лиц (телефоны, почту, ФИО клиентов).»

### Шаг 13. Строка в политику
В `policy.html` добавь пункт:
> «Для предварительной оценки сметы введённый текст обрабатывается языковой моделью YandexGPT (Yandex Cloud, РФ). Текст не сохраняется. Не вводите персональные данные третьих лиц.»

---

## Часть 5. Финальная проверка

### Шаг 14.
1. Открой сайт с опубликованным `js/ai-config.js`.
2. Выбери проект «Другое» → кнопка **✨ Собрать из брифа**.
3. Вставь: «Презентация на 15 слайдов для стартапа, есть графики и схемы».
4. Нажми **Рассчитать**.
   ✅ Должны появиться этапы с часами, а сумма — посчитаться автоматически.
   ✅ Выдерни интернет и повтори — должен сработать запасной регексп-вариант (без ошибки).

## Сколько будет стоить
YandexGPT Pro ≈ 0,2–0,8 ₽ за один разбор брифа. Тысяча смет ≈ несколько сотен рублей. Lite — ещё дешевле.

## Чек-лист безопасности (закрывает 152-ФЗ)
- [x] Данные остаются в РФ (YandexGPT) — нет трансграничной передачи.
- [x] Персональные данные вырезаются в функции (`stripPII`).
- [x] Функция ничего не хранит (нет базы ПД).
- [x] Предупреждение пользователю + строка в политике.
- [ ] Финальные формулировки политики показать юристу (из-за размеров штрафов — недорогая страховка).
